# 微信公众号科普知识写作 Skill

## 风格定位

**学习对象：**
- 果壳网 - 有趣有料有态度
- 科学松鼠会 - 科普也可以很有趣
- 博物杂志 - 知识 + 趣味

**核心原则（必须遵守）：**
1. **科学准确** - 数据、概念、原理必须准确
2. **通俗易懂** - 用生活化语言解释专业概念
3. **来源可靠** - 引用权威期刊、官方数据
4. **避免误导** - 不夸大、不绝对化

**核心特点：**
- ✅ 准确：科学事实准确无误
- ✅ 通俗：外行也能看懂
- ✅ 有趣：不枯燥，有吸引力
- ✅ 严谨：区分事实与推测

## 内容要求

### ✅ 必须做到的

1. **数据来源**
   - 优先引用权威期刊（Nature、Science 等）
   - 官方机构数据（中科院、卫健委等）
   - 注明数据来源和发布时间

2. **概念解释**
   - 专业术语必须解释
   - 用比喻帮助理解
   - 避免过度简化

3. **表述严谨**
   - 区分"研究表明"和"已证实"
   - 避免"绝对"、"一定"等词
   - 说明研究局限性

### ❌ 严格禁止的

1. **伪科学**
   - 不传播未经证实的理论
   - 不宣传伪科学产品
   - 不夸大科学作用

2. **误导性表述**
   - "科学家说"（无来源）
   - "研究表明"（无文献）
   - "绝对有效"（绝对化）

3. **过时信息**
   - 使用最新研究成果
   - 标注信息时效性
   - 及时更新修正

## 文章结构

```
【开头】悬念引入（100-150 字）
- 生活中的现象/问题
- 引发好奇心的提问
- 简洁有力的核心观点

【正文】4-5 个部分（每部分 400-500 字）
1. 现象描述（具体场景）
2. 科学原理解释（通俗易懂）
3. 研究数据支撑（权威来源）
4. 实际应用/影响（生活相关）
5. 常见误区澄清（如有）

【结尾】总结延伸（150-200 字）
- 核心知识点总结
- 实用建议/启示
- 延伸阅读方向
```

## 语言特色

### ✅ 推荐的表达
- "研究表明..."（附文献）
- "用通俗的话说..."
- "可以这样理解..."
- "简单来说..."
- "数据显示..."
- "专家解释..."

### ❌ 避免的表达
- "据说..."（无来源）
- "绝对..."（绝对化）
- "肯定..."（太绝对）
- "所有人都知道..."（以偏概全）
- 过度使用专业术语

## 参考文献格式

**推荐格式：**
```
[1] 作者。论文标题。期刊名，年份，卷 (期): 页码.
[2] 机构名称。报告标题。发布年份.
[3] 官方网站。页面标题。访问日期.
```

**示例：**
- ✅ "据《自然》杂志 2023 年的一项研究..."
- ✅ "中科院物理所解释称..."
- ✅ "国家卫健委数据显示..."

- [ ] 所有数据都有来源
- [ ] 专业术语已解释
- [ ] 无绝对化表述
- [ ] 无伪科学内容
- [ ] 信息来源权威可靠
- [ ] 无误导性内容

## 编程接口

```python
from wechat_science_writer import WechatScienceWriter

writer = WechatScienceWriter()

# 生成科普文章
article = writer.write(
    topic="为什么天空是蓝色的",
    style="explanation",  # explanation/news/myth
    word_count=3000
)

# 验证内容准确性
check_result = writer.verify_content(article)
if check_result["is_valid"]:
    print("✅ 内容验证通过")
```

1. **科学准确性第一** - 宁可少说，不可错说
2. **通俗易懂** - 让外行看懂是本事
3. **严谨表述** - 区分事实与观点
4. **持续学习** - 科学知识更新快
5. **承认未知** - 不知道就说不知道
